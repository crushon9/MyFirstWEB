리눅스 (Linux)

리눅스는 1991년 9월 17일 리누스 토르발스가 처음 출시한 운영 체제 커널인 리눅스 커널에 기반을 둔 오픈 소스 유닉스 계열 운영 체제 계열이다.

커널 (kernel)

하드웨어를 통제하는 핵심 코어 운영체제. 쉘에게 전달받은 사용자의 요청을 하드웨어에게 전달

쉘 (shell)

사용자 명령어들을 컴퓨터 언어로 변역하여 커널에 전달. shell은 기본적으로 bash, tcsh, zsh 등이 사용되며 window에서는 cmd를 기반으로 사용되고있다.

터미널 (Terminal)

터미널이란 컴퓨터와 사용자간의 서로 소통시켜주는 인터페이스(번역기와 같은느낌)라고 할 수 있다.인터페이란 텍스트 기반과 그래픽 기반(GUI) 두가지를 기반을 포함하는 용어이지만 리눅스 터미널은 텍스트 기반을 말한다. 터미널은 컴퓨터와 소통하기 위해서 키보드를 사용하는것을 말한다. 터미널을 어떻게 만들기에 따라서 컴퓨터간의 소통도 가능하다. 최근에는 리눅스에도 GUI기반이 잘되어있어서 리눅스 명령어를 몰라도 리눅스를 사용할 수 있다.

Process와 Program

  • Processor : CPU (Process를 처리하는 장치)
  • Memory : RAM
  • Storage : HDD, SDD
  • Program : 프로그래밍 언어를 사용하여 수행절차를 표현해 놓은 명령어들의 집합, 컴퓨터에서 어떤 작업을 위해 실행할 수 있는 '정적인 상태'의 파일
  • Process : 프로그램이 실행되어 돌아가고 있는 상태, 컴퓨터에서 연속적으로 실행되고 있는 '동적인 상태'의 컴퓨터 프로그램
  • 리눅스의 command는 Storage(HDD,SDD)에 Program파일로 저장되어있다가, 실행하면 Memory(RAM)에 적재되고, CPU에 의해 처리되는데 이 상태가 Process다.

CLI 와 GUI

인터페이스(interface)는 서로 다른 두 개의 시스템, 장치 사이에서 정보나 신호를 주고받는 경우의 접점이나 경계면을 뜻한다.즉, 사용자가 기기를 쉽게 동작시키는 데 도움을 주는 시스템을 의미한다. Linux의 인터페이스는 CLI 방식이다.

  • CLI (Command Line Interface) : 1980 년대에 사용 된 전통적인 인터페이스로 터미널이나 콘솔 창에 사용자가 명령을 작성하여 운영 체제와 통신 하는 인터페이스이다. 사용자는 작업을 수행하기 위해 명령 또는 명령 트레인을 입력해야하는 단점이 있지만, 컴퓨터 용량, CPU, 메모리를 적게 차지하는 장점이 있다. GUI가 할 수 있는 모든 작업은 CLI로 할 수 있다.
  • GUI (Graphical User Interface) : 그래픽(이미지, 아이콘, 텍스트 상자와 같은 시각 보조 도구)을 사용하여 사용자가 운영 체제 또는 응용 프로그램과 상호 작용하는 인터페이스이다. 사용자에게 친숙하지만 무겁고, 키보드, 마우스, 모니터와 같은 컴퓨터를 조작하기 위한 입력과 출력 장치를 필요로 한다.

디렉토리 (Directory)

디렉토리란 파일 저장소를 의미하며, 리눅스 디렉토리는 최상위 디렉토리를 기준으로 하위 디렉토리들이 존재하는 계층적 트리 구조로 구성되어 있다.

Terminal 명령어

명령어 뒤에 -옵션(parameter)을 붙일 수 있다

pwd (print working directory)

현재 위치하고 있는 디렉토리를 알려주는 명령어
리눅스의 명령어는 현재위치에서 명령이 내려지므로 위치를 파악하는것이 매우 중요하다

mkdir (make directory)

현재위치에 디렉토리를 생성하는 명령어

  • mkdir + 디렉토리명

cd (change directory)

디렉토리를 이동하는 명령어

  • cd + 이동할 디렉토리의 경로
  • cd .. 부모 디렉토리로 이동 (.은 하위)
  • cd ~ home 디렉토리로 이동

상대경로는 현재 디렉토리의 위치를 기준으로 다른 디렉토리의 위치를 표현하는 것이고
절대경로는 최상위 디렉토리인 루트디렉토리(/)를 기준으로 경로를 표현하는 것이다

ls (list segments)

현재 디렉토리의 파일 목록을 출력하는 명령어

  • ls -l : 파일정보 자세히
  • ls -a : 숨긴파일도 모두 (숨긴파일은 .로 표기)
  • ls -al : l과 a 옵션함께

rm (remove)

파일이나 디렉토리를 삭제하는 명령어

  • rm + 파일명
  • rm -r + 디렉토리명

cp (copy)

파일이나 디렉토리를 복사하는 명령어

  • cp + 파일명 + 이동할경로
  • cp -r + 디렉토리명 + 이동할경로

이동시킬 파일(디텍토리)이 명령을 내리는 디렉토리 안에 없을때는 경로를 포함하여 적어준다

mv (move)

파일이나 디렉토리를 이동하는 명령어

  • mv + 파일명 + 이동할경로
  • mv -r + 디렉토리명 + 이동할경로

touch

파일을 생성하거나 갱신하는 명령어

  • touch 이름.txt

cat (concatenate)

파일 또는 표준입력 내용을 그대로 표준출력에 출력하는 명령어

  • cat 파일이름

--help

해당명령의 간단한 사용팁이 출력된다

  • 명령어 --help

man (manual)

특정 명령어의 상세한 사용설명서로 이동한다

  • man + 명령어
  • /검색어
  • n 이동키
  • q 페이지종료

clear

화면의 내용을 모두 초기화

Sudo (super user do)

관리자(super user /root)의 권한{permission}으로 명령을 실행 할 때 사용. 모든 사용자가 이 명령어를 적용시킬수 있는 것은 아니다

find

실시간 디렉토리에 저장된 파일을 검색해줌

  • find + / + -name + *.txt : root 디렉토리부터 파일이름으로 검색
  • find + . + -name + *.txt : 현재디텍토리의 하위디렉토리부터 파일이름으로 검색

locate

데이터베이스(mlocate)에 저장된 파일정보를 검색하여 결과를 출력해줌

  • locate + *.txt
  • sudo updatedb : mlocate 데이터베이스를 업데이트하는 명령어

whereis

검색어가 어느 디렉토리에 있는지 주소를 알려줌
ls라는 명령어는 /bin 디렉토리에 저장된 프로그램인데, 어떻게 다른 모든 디렉토리에서 실행이 가능한 걸까?
$PATH라는 환경변수에 기본경로가 담겨있기때문에 가능하다. ($PATH의 경로들은 편집이 가능하다.)

grep (global / regular expression / printprint)

파일의 문자열에서 찾고자하는 검색어가 포함된 행을 출력

  • grep + 검색어 + 파일

alias (별명)

명령어에 단축키를 지정할수있다

  • alias 단축키 = '원래명령어'

Redirection 과 Pipe

컴퓨팅에서 표준 스트림을 사용자 지정 위치로 우회할 수 있는 다양한 유닉스 셸을 포함한 대부분의 명령어 인터프리터에 일반적인 명령이다. 방향 지정, 방향 변경으로도 부른다.(IO Stream 통제)

    Redirection
  • 명령어1 > 파일1 명령어1을 실행한 출력물을 파일1로 내보낸다. 파일1이 존재할땐 기존내용은 지우고 덧씌운다.
  • 명령어1 >> 파일1 파일의 끝에 출력물을 추가한다.
  • 명령어1 < 파일1 명령어1을 실행하되, 파일1이 입력의 대상이 된다.
  • 명령어1 < 들어오는파일> 나가는파일 두 개의 기능을 수행. 명령어1은 들어오는파일에서 내용을 불러들인 뒤 나가는파일에 기록한다.
  • 명령어 1> 파일a 2> 파일b 표준출력일때 파일a에 Redirection되고 표준오류일때 파일b에 Redirection된다

    Pipe
  • Pipe (A | B) A 커맨드의 표준 출력을 B 커맨드의 표준 입력으로 사용한다

명령접속사

  • ; 앞의 명령어가 실패해도 다음 명령어가 실행
  • && 이전 명령어의 실행결과가 참(true)일 때만 다음 명령을 실행
  • & 앞의 명령어를 백그라운드로 돌리고 동시에 뒤의 명령어를
  • { } 명령을 그룹핑

Package Manager

패키지매니저를 통해 소프트웨어를 간단하게 설치할 수 있다. 다양한 패키지매니저 중에 apt 사용방법을 알아보자

    실행방법
  • sudo apt-get update; → 최신목록업데이트
  • sudo apt-cache search + 검색어; → 검색
  • sudo apt-get install + 프로그램이름; → 설치
  • sudo apt-get upgrade + 프로그램이름; → 특정 프로그램 업그레이드
  • sudo apt-get upgrade; → 전체 프로그램 업그레이드
  • sudo apt-get remove + 프로그램이름; → 삭제

wget

    실행방법
  • wget -O 저장할이름 url주소

htop

현재 실행중인 프로세스 목록을 보여주는 프로그램

    실행방법
  • sudo htop
  • sudo kill PID(프로세스식별숫자)
  • Load average 부하평균으로 CPU부하 정도를 알 수 있다

파일 색상 의미

  • 흰색 : 일반파일

  • 녹색 : 실행파일

  • 빨간색 : 압축파일

실행 (Execute)

백그라운드 실행

실행중인 프로그램을 종료하지 않고, 백그라운드로 보낸 뒤 다른 프로그램을 실행하고자 할때

  • Ctrl + z : 실행중인 프로그램을 백그라운드로 보내는 단축키
  • jobs : 백그라운드 프로그램의 목록을 출력하는 명령어
  • fg : 백그라운드 프로그램 중 우선순위(+,-,)대로 foreground로 전환하는 명령어
  • fg %숫자 : 백그라운드 프로그램 중 지정한 프로그램 foreground로 전환
  • kill %숫자 : 지정한 프로그램 종료
  • 명령어+& : 해당 명령어가 실행될 때 백그라운드로 실행 (오래걸리는 프로그램을 실행할때 사용)

항상 실행 (damon)

항상 실행되는 프로그램을 demon이라고 한다. 예를들면 서버프로그램인 아파치같은

  • sudo service + 프로그램 start : 항상실행으로 프로그램을 실행

정기적 실행

알람이나 백업 등 정기적으로 처리해야할 작업은 cron이라는 프로그램을 이용한다

  • crontab -e : 크론에디터로 이동되어 정기실행 작업을 편집할 수 있다

쉘 시작시 자동실행 (Startup script)

쉘이 시작될 때 실행되는 명령을 편집하여 쉘을 커스터마이징 할 수 있다.

  • bashrc 파일에 접속하여 시작프로그램을 편집할 수 있다(쉘이 bash가 아니라면 다른파일)

다중사용자

유닉스 계열 운영체제는 여러 명이 함께 사용할 수 있는 기능을 가지고 있다. 이 기능을 사용하면 권한과 보안의 문제가 발생할 수 있다.

관리자와 일반사용자

Super user, Root user(#)

  • 홈디렉토리는 /root 이다 (최상위디렉토리 / 를 루트디렉토리라고 하며, 그 하위의 root 디렉토리)
  • su - root : 현재 사용자를 root사용자로 변경 (모든사용자가 가능한건 아님)
  • sudo passwd -u root : 관리자모드 접근 잠금해제 (우분투환경은 잠금해제 후 관리자로 변경가능)
  • sudo passwd -l root 관리자모드 접근 잠금

user($)

  • 홈디렉토리는 /home 이다

사용자 식별 명령어

  • id : 명령내린 사용자의 정보를 출력
  • who : 현재 접속한 사용자의 목록을 출력
  • ~ : 현재 사용자의 홈디렉토리를 가리키는 특수기호

사용자 추가

  • sudo useradd -m 이름 : 엔터 후 현재 명령내린 사용자의 passwd를 입력하면 사용자추가
  • sudo passwd 이름 : 사용자의 passwd를 설정(한 글자로)

권한 (permission)

[파일, 디렉토리]에 대해 [read, write, excute]하는 것을 설정할 수 있다. (디렉토리에서 r은 열람, w은 파일생성삭제, x는 cd를통한접근을 의미한다.) 대상 사용자로는 [owner, group, ohter]가 있으며, 그룹은 생성편집이 가능하다

    왼쪽부터 순서대로
  • - : type, 파일은- 디렉토리는d
  • rw-rw-r-- : access mode for owner/group/ohter, rwx(read/write/excute)
  • 1 : 링크수
  • ubuntu ubuntu : owner group 이름
  • 3 : 크기
  • Mar 24 00:04 : 최근수정일
  • permission.txt : 이름

chmod (change mode)

파일, 디렉토리의 permission을 변경하는 명령어

  • chmod [u오너g그룹o아더a모두][-+=][rwx] 대상이름
  • -R : 대상디렉토리 하위의 모든파일에 적용
  • rwx 경우의수는 8가지인데, 0~7의 숫자를 통해 명령을 내릴수도 있다(octal mode)

네트워크 (Network)

  • client & server : 정보를 요청하고 제공받는 관계
  • host : 네트워크에 연결된 컴퓨터나 기타장치(클라이언트와 서버 각각의 컴퓨터)
  • network : host가 그물망처럼 연결되어 통신하는 형태
  • internet : 인터네트워크(inter-network)'라는 말에서 시작되었으며, 전 세계 컴퓨터들을 하나로 연결하는 거대한 컴퓨터 통신망

Server program

웹브라우저 프로그램이 크롬,인터넷익스플로러 등 다양한 것처럼 웹서버 운영 프로그램도 다양하다. 그중 apache 가 대중적이다.

Address

MAC (Media Access Control Address)

네트워크 세그먼트의 데이터 링크 계층에서 통신을 위한 네트워크 인터페이스에 할당된 고유 식별자 즉, 컴퓨터의 물리적 주소

IP address (Internet Protocol address)

IP 주소는 컴퓨터 네트워크에서 장치들이 서로를 인식하고 통신을 하기 위해서 사용하는 특수한 번호이다.

    public address와 private address
  • 통신사에서 부여하는 public address와 공유기때문에 부여되는 private address가 다른 경우가 많다
  • ip addr : 내선번호와 같은 private address ip를 출력하는 명령어
  • curl ipinfo.io/ip : 외부에 접속할때 사용하는 public address ip를 출력
  • ip 127.0.0.1 / domain localhost : 하나의 컴퓨터로 서버와 클라이언트를 수행할때 자기자신을 가르키는 특별한주소

Domain name

도메인 네임은 넓은 의미로는 네트워크상에서 컴퓨터를 식별하는 호스트명을 가리키며, 좁은 의미에서는 도메인 레지스트리에게서 등록된 이름을 의미한다. domain name과 ip address는 전화번호부에 저장된이름과 전화번호와 같은 관계이다.
DNS(Domain Name Server)에는 거대한 전화번호부와 같이 모든 ip의 domain이 저장되어있다.
ip 127.0.0.1 / domain localhost : 하나의 컴퓨터로 서버와 클라이언트를 수행할때 자기자신을 가르키는 특별한주소

URL(Uniform Resource Locator)

인터넷에서 리소스의 위치를 가리키는 문자열 https:// 뒤의 도메인 이름 모두 포함한 경로

Port

포트는 네트워크가 드나다니는 연결구멍 같은것인데 65000여개가 있고, 0~1024포트는 용도에 따라 표준으로 지정되어있다. 그중 웹은 :80을 표준으로한다. 하지만 다수에게 알려지길 원하지 않을때 포트번호를 지정할수있다.

port forwarding

포트포워딩은 컴퓨터 네트워크 상에서 패킷이 라우터와 같은 네트워크 게이트를 지날 때 IP 주소와 포트 번호 결합의 통신 요청을 라우터에 내선으로 연결된 컴퓨터등을 지정하여 넘겨주는 네트워크 주소 변환의 응용법이다.

SSH (Secure Shell)

시큐어 셸은 네트워크 상의 다른 컴퓨터에 로그인하거나 원격 시스템에서 명령을 실행하고 다른 시스템으로 파일을 복사할 수 있도록 해 주는 응용 프로그램 또는 그 프로토콜을 가리킨다.
리눅스에서 ssh 명령을 사용하여 외부컴퓨터에서 원격으로 서버컴퓨터에 접근하여 제어할 수 있다.

ssh key

ssh key는 public key와 private key로 이루어져 있는 비대칭적 암호화(RSA) 보안키이다.
keygen 명령을 통해 파일을 생성하여, 원하는 컴퓨터에 복사해두면 로그인 없이 로그인이 가능하다. 이 기능을 이용하여 rsync로 로그인없이 정기적으로 백업이 가능하다.

동기화 (Synchronization)

동기화는 시스템을 동시에 작동시키기 위해 여러 사건들을 조화시키는 것을 의미한다.
리눅스에서 rsync(remote sync) 명령을 사용하여 폴더 및 파일을 동기화 시킬수 있다.
